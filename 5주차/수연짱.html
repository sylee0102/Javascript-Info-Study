📌  프라미스와 에러 핸들링

프라미스가 거부되었을 때 rejection 핸들러로 넘어가기 때문에 프라미스 체인을 사용하면 에러 처리가 가능

  • successHandler는 프라미스가 성공적으로 이행(resolve)되었을 때 실행되는 함수입니다.
  • rejectionHandler는 프라미스가 거부(reject)되었을 때 실행되는 함수입니다.

[수연이 설명]

🍦가정 : 아이스크림 가게에서 아이스크림을 주문했다

  1. 프라미스는 약속 : 아이스크림 가게 주인이 “곧 아이스크림을 만들어 줄게요!”라고 약속

    👉🏻 이거시 바로 프라미스다!!

  1. 좋은 결과 (성공) : 아이스크림이 잘 만들어지면, 명와님은 마싯게 먹을 수 있자나여?

    👉🏻 이거슨 프라미스가 잘 지켜진거예요!!

  1. 나쁜 결과 (실패) : 하지만 가끔 문제가 생길 수 있쟈냐? 재료가 다 떨어졌다면?

    👉🏻 이거슨 프라미스가 잘 지켜지지 못한거예요!!

  1. 에러 핸들링 : 만약 아이스크림을 못 받게 되면? 돈 와리! “에라 핸들러” 라는 특별한 계획을 세움

    👉🏻 다른 맛의 아이스크림 받거나 / 주스로 바꾸거나 / 돈 돌려받기

    그러면,,,, 당황하지않고, 대처가 가능하겠지 뭐,,,

  1. 프라미스 체인 : 명와님이 아이스크림 받고, 촤카릿 시럽을 뿌려달라고 할 수 있져

    👉🏻 여러단계를 연결하는 것을 후라미스 체인이라고 함

📌  프라미스 API

Promise 클래스의 5가지 정적 메서드

⬇️ “가장 많이 사용하는 메서드” ⬇️

  • 모든 프라미스가 이행될 때까지 기다렸다가 그 결과값을 담을 배열을 반환
  • 주어진 프라미스 중 하나라도 실패하면 promise.all은 거부, 나머지 프라미스의 결과는 무시

  • 최근에 추가 된 메서드
  • 모든 프라미스가 처리 될 때까지 기다렸다가 그 결과를 담은 배열을 반환함
  • 객체엔 아래와 같은 정보가 담김
    • status : "fulfilled" 또는 "rejected"
    • value (프라미스가 성공한 경우) 또는 reason (프라미스가 실패한 경우)

  • 가장 먼저 처리 된 프라미스의 결과 또는 에러를 담은 프라미스를 반환

  • 주어진 값을 사용해 이행 상태의 프라미스를 만듦

  • 주어진 에러를 사용해 거부 상태의 프라미스를 만듦

프라미스화는 마법 지팡이를 사용하는 것과 비슷해요~

  1. 마법 지팡이 (프라미스화) : 명와님이 평범한 장난감을 가지고 있는데, 이 장난감은 가만히 있는거예요
    근데 마법 지팽이를 가지고 있는 진님이 싸악— 치며는 이제 특별한 힘을 가지고 있는 장난감으로 변하는데 이게
    프라미스화인겁니다!!
  1. 특별한 힘 (프라미스의 특징) :
    • 진님의 마법의 장난감은 “나 준비됐어! 또는 “앗, 문제가 생겼어”라고 말할 수 있음
    • 또, 진님의 잰낸갬은 동혁이의 매뱁 잰낸갬과 함께 놀 수 있음 (체이닝)
    • 문제가 생기면 해결사인 지현님을 바로 부를 수 이쨔냐 (에라 처리)
  1. 예시 : 수연이의 평범한 주주시크릿이 가만히 있는데, 진님의 마법 지팽이로 툭— 치며는 이제 :
    • 쥬쥬 씨꼬리또가 걸을 수 있게 됨 (작업 시작)
    • 걷다가 넘어지면 바로 일어날 수 있게 됨 (에라 처리)
    • 걷기가 끝나면 춤을 출 수도 있음 (체이닝)
  1. 왜 좋을까요?
    • 명와님의 특별해진 장난감과 수연이의 쥬쥬시끄리또 장난감은 서로 잘 협력해여 (비동기 작업 관리)
    • 문제가 생겨도 당황하지 않고 해결할 수 있음 (에라 처리 용이)
    • 여러가지 재미있는 일을 연달아 할 수 있어요 (체이닝 가능)
  1. 실제 생활에서는 : 컴퓨터 프로그램에서 시간이 좀 걸리는 일을 할 때, 이 일을 프라미스화해서 더 잘 관리할 수 있음

📌  마이크로테스크

프라미스 핸들러 .then .catch .finally는 항상 비동기적으로 실행

프라미스가 즉시 이행되더라도 .then .catch .finally 아래 있는 코드는 이 핸들러들이 실행되기 전에 실행 됨 

let promise = Promise.resolve();

promise.then(() => alert("프라미스 성공!"));

alert("코드 종료");
  1. alert 창이 가장 먼저 뜸
  1. “프라미스 성공!”이 나중에 출력

마이크로태스크 큐

비동기 작업을 처리하려면 적절한 관리가 필요해요~~~

그러기 위해서는 ECMA에서 PromiseJobs라는 내부 큐를 명시함 V8 엔진에서는 이를 ‘마이크로태스크 큐’라고 부르기 때문에 이 용어가 좀 더 선호됨

명세서 설명

  • 마이크로태스크 큐는 먼저 들어온 작업을 먼저 실행
  • 실행할 것이 아무것도 남아있지 않을 때만 마이크로태스크 큐에 있는 작업이 실행되기 시작

“프라미스 성공!”을 먼저 출력되게 하려면? 
let promise = Promise.resolve();

promise.then(() => alert("프라미스 성공!"));

alert("코드 종료");

.then을 사용해 큐에 넣기!!
